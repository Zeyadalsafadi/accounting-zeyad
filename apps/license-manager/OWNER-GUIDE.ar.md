# دليل استخدام License Manager

هذا الدليل مخصص لمالك النظام فقط.

الأداة محلية، وتستخدم لإدارة مفاتيح التوقيع وإصدار وتجديد والتحقق من التراخيص الخاصة بنظام `دكانتي Shop`.

## تشغيل الأداة

من جذر المشروع:

```powershell
npm run dev:license-manager
```

بعد التشغيل:

- واجهة الأداة: `http://localhost:5174`
- الخدمة المحلية: `http://localhost:4174`

## 1. إعداد المفاتيح لأول مرة

اذهب إلى شاشة `Keys`.

يوجد حالتان:

### الحالة الأولى: لا توجد مفاتيح سابقة

1. اكتب اسمًا للمفتاح مثل: `Primary Owner Key`
2. اضغط `Generate Key Pair`

سيتم إنشاء:

- مفتاح خاص `Private Key`
- مفتاح عام `Public Key`

وسيتم حفظهما محليًا على جهازك فقط.

### الحالة الثانية: لديك مفاتيح قديمة

إذا كنت أصدرت تراخيص سابقًا وتريد أن تبقى التراخيص القديمة والجديدة متوافقة:

1. الصق `Private Key PEM`
2. الصق `Public Key PEM`
3. اضغط `Import Keys`

## 2. ربط المفتاح العام مع الـ API

من شاشة `Keys`:

1. اضغط `Copy .env`
2. ضع الناتج في إعدادات API

مثال:

```env
LICENSE_PUBLIC_KEY=-----BEGIN PUBLIC KEY-----\n...\n-----END PUBLIC KEY-----
LICENSE_ENFORCEMENT=strict
```

مهم:

- السيرفر يحتاج المفتاح العام فقط
- لا تضع المفتاح الخاص داخل `.env`

## 3. إصدار ترخيص جديد

اذهب إلى شاشة `Issue License`.

املأ الحقول التالية:

- `Customer name`
- `License ID`
- `Plan`
- `Issued at`
- `Expires at`
- `Grace days`
- `Max devices`
- `Enabled modules`
- `Internal notes`

إذا أردت رقم ترخيص جديد تلقائيًا:

- اضغط `Generate License ID`

إذا اخترت خطة مثل `STANDARD` أو `PRO`:

- سيتم تعبئة الوحدات والإعدادات الافتراضية الخاصة بالخطة

بعد التأكد من `Payload Preview`:

- اضغط `Issue License`

الناتج:

- توكن الترخيص النهائي
- حفظ السجل داخل قاعدة البيانات المحلية
- إمكانية النسخ أو التصدير

## 4. ماذا أرسل للعميل؟

أرسل للعميل:

- التوكن النهائي فقط

لا ترسل للعميل:

- المفتاح الخاص
- المفتاح العام
- ملفات PEM
- قاعدة البيانات المحلية

## 5. استعراض التراخيص

اذهب إلى شاشة `Issued Licenses`.

يمكنك من خلالها:

- البحث باسم العميل
- البحث برقم الترخيص
- الفرز حسب الإصدار أو الانتهاء
- معرفة الحالة الحالية
- فتح التفاصيل الكاملة

وعند فتح السجل ستشاهد:

- بيانات الترخيص
- التوكن الكامل
- `Payload`
- عدد الأيام المتبقية
- نهاية فترة السماح
- تاريخ السجل والإصدارات المرتبطة به

## 6. تجديد ترخيص

من شاشة `Issued Licenses`:

1. افتح الترخيص المطلوب
2. اضغط `Renew`
3. عدّل تاريخ الانتهاء أو الخطة أو الوحدات إذا لزم
4. اضغط `Issue License`

سيتم إنشاء سجل جديد مع الاحتفاظ بالسجل السابق.

## 7. إعادة إصدار ترخيص

إذا احتجت إعادة إصدار نفس الترخيص أو تعديل بسيط مع الاحتفاظ بالأثر:

1. افتح السجل
2. اضغط `Reissue`
3. راجع البيانات
4. اضغط `Issue License`

سيتم تسجيل العملية كسجل جديد ضمن التاريخ.

## 8. التحقق من أي توكن

اذهب إلى شاشة `Validate License`.

1. الصق التوكن
2. اضغط `Validate Token`

ستظهر لك:

- هل الصيغة صحيحة
- هل التوقيع صحيح
- حالة الترخيص
- البيانات المفكوكة من التوكن

## 9. الإعدادات

من شاشة `Settings` يمكنك ضبط:

- مسار حفظ المفاتيح
- عدد أيام السماح الافتراضي
- عدد الأيام التي تعتبر الترخيص قريب الانتهاء
- بادئة رقم الترخيص
- قوالب الخطط

## 10. أين يتم التخزين؟

بشكل افتراضي يتم حفظ:

- قاعدة البيانات المحلية
- ملفات المفاتيح

داخل مجلد بيانات المستخدم المحلي على جهازك.

يمكنك تغيير مسار حفظ المفاتيح من شاشة `Settings`.

## 11. قواعد الأمان

يجب الالتزام بهذه القواعد دائمًا:

- احتفظ بالمفتاح الخاص على جهاز المالك فقط
- لا تضع المفتاح الخاص داخل المستودع
- لا ترسل المفتاح الخاص للعميل
- لا تشحن المفتاح الخاص مع التطبيق
- العميل يحصل على التوكن النهائي فقط

## 12. أسرع سيناريو عمل يومي

عند إصدار ترخيص جديد:

1. افتح `Issue License`
2. أدخل اسم العميل
3. اختر الخطة
4. حدد تاريخ الانتهاء
5. اضغط `Issue License`
6. اضغط `Copy Token`
7. أرسل التوكن للعميل
